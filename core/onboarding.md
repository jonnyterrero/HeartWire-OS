# Onboarding — How to Use HeartWire OS

This document teaches you (or an AI assistant) how to navigate the HeartWire OS workspace.

- **Class notes** → [`learning/coursework/`](../learning/coursework/)
- **Self-study** → [`learning/self-study/`](../learning/self-study/)
- **Cheatsheets** → [`learning/cheatsheets/`](../learning/cheatsheets/)
- **Homework / problem sets** → [`learning/problem-solving/`](../learning/problem-solving/)
- **Free-form notes & writing** → [`learning/notes/`](../learning/notes/)
- **Apps, hardware, engineering projects** → [`build/projects/`](../build/projects/) · [`build/apps/`](../build/apps/) · [`build/hardware/`](../build/hardware/)
- **Product / brand prototypes** → [`build/prototypes/`](../build/prototypes/)

- **AI automations & workflows** → [`system/automations/`](../system/automations/)
- **AI agents** → [`system/agents/`](../system/agents/)
- **Integrations (Notion, Supabase, GitHub, etc.)** → [`system/integrations/`](../system/integrations/)
- **Data / ingest / ETL pipelines** → [`system/pipelines/`](../system/pipelines/)
- **PDFs, datasets, papers** → [`resources/`](../resources/)
- **Deprecated / archived** → [`archive/`](../archive/)

## AI agent instructions

Agents should:

- read folder `README.md` files to understand context before acting
- follow the category structure to locate information and place new files
- store new notes, summaries, solutions, and project files under the correct directory
- prefer lowercase kebab-case for new files and folders
- never write into `archive/legacy-snapshots/` — that is a read-only quarantine for old structures

## Philosophy

Everything in HeartWire OS should be:

- organized
- searchable
- useful
- actionable
- future-proof
